# Jinguzhou 0.3.0-beta Agent Integration Plan

This beta release turns the alpha adapter foundation into framework-facing
entry points.

## Scope

- MCP adapter helper module
- LangChain-style tool middleware
- runnable examples and release validation

## Step 1: MCP Adapter

Deliverables:

- `jinguzhou.adapters.mcp.MCPToolAdapter`
- `normalize_mcp_tool_call(...)`
- `build_mcp_tool_call_request(...)`
- example under `examples/mcp-tool-security/`

Acceptance:

- MCP JSON-RPC `tools/call` payloads normalize into `NormalizedToolCall`
- extracted arguments use existing file/network/database policy packs
- example blocks a metadata endpoint request before execution

## Step 2: LangChain Middleware

Deliverables:

- `jinguzhou.integrations.langchain.JinguzhouToolMiddleware`
- `guard_tool(...)`
- `ToolPolicyViolation`
- example under `examples/langchain-tool-policy/`

Acceptance:

- blocked and review-required tool calls do not execute
- safe tool calls continue to execute
- sync and async invocation paths are covered by tests

## Step 3: Release Validation

Deliverables:

- tests for MCP and LangChain integration helpers
- release validation entries for both runnable examples
- documentation and release notes updated
